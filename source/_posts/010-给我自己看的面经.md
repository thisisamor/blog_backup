## 杂七杂八的问题总结

### a - micro-processor

#### 笔试

第 N 个二进制数中出现了多少个数字 1，然后录视频解释自己的解题思路。

<small style="opacity: 0.7;"> 
应该解对了，事后仔细想一想好像数据范围考虑不全。用的 int 数据类型，会溢出。

感觉讲解的时候思路不够清晰，应该应用的不是最好的方法，后来感觉自己用的是 divide & conquer 的思路，但当时还不知道这个名词，然后没提到复杂度的事。
</small>

#### 一面

- 解释 CPU pipelined 设计原理
- 解释 cache 的用处（hit / miss）
- 解释 virtual memory

<small style="opacity: 0.7;"> 当时刚学，vm 几乎啥也不会，照猫画虎地背书，也没提自己的啥理解。</small>

### G - software 

#### 笔试

两道 coding 题，限时，算法非常简单。

#### 一面

- 口头讲从一个 array 找符合要求的数对的方法，是不是一直都正确，有没有更好的方法。
- 给看一段 java 代码，让讲代码是干什么的（检查回文 - palindrome），然后改错，优化的方法，改更合适的变量名。
- 关于某个官网上 project 的讨论。

<small style="opacity: 0.7;"> 面试官很在意是否想要在软件领域长久发展（以及在他们公司长久发展），我不是那么 convinced，感觉自己也不是很有激情。第一个问题一开始理解错了意思，复杂度超级高，花了半天和面试官说明自己的思路，感觉他和我都满头问号，后来明白题意才豁然开朗，真的超简单的问题。感觉可能本来应该问三个问题的。二面回复等的比较久，每天都觉得要凉了。</small>

#### 二面

HR 面，主要就是 cultual talk. 
- 最艰难的一次经历
- 团队合作的重要性
- 和不好合作的人合作的经历
- 你为面试做了哪些准备
- 你的朋友是怎么评价你的
- 你认为什么是成功
- 你希望面试后我记住你的哪三件事
- 公司和岗位介绍

<small style="opacity: 0.7;"> 都是挺经典的 HR 面试题目。不过真的第一次面对面说这些话题的感觉，和对着文字在脑子里想的感觉完全不一样。面试官超漂亮，也很温和，我大脑一片空白，但是没怎么卡壳。比起面试，体验更像雅思口语。</small>

#### 三面

- 机器人按要求走路，用给的玩具语言编程（有点像汇编语言），考虑限制条件，考虑 corner case。其实不是很难，但是我一直脑子里忍不住冒出来走迷宫的小车，想象中的逻辑智能了很多，结果中间卡住了。
- python 题，解释一些简单的程序怎么运行，然后自己动手写几道题，结合实际，讨论怎么优化。比较有趣的有找 anagram（字母一样但词不一样），还有 0-9 的整数序列快排。
- 和项目主管谈，比想象中久，内容和 HR 聊的差不多。

### F - c++

#### 笔试

一堆没啥实际用处的选择题。
- 概念性的选择题，文字描述
- 数据结构优缺点、问题等

四道填空题。
- 形如海盗分硬币之类的问题。

两道编程题。
- SQL，比较基本的 SELECT-FROM-WHERE 加若干条件就能解决。
- 输入一个字符串 a 和另一个字符串 b，增加一个字母需要花费 x，删去一个字母需要 y，替换一个字母需要 z，改动的时候字符串可以自动补位，计算把 a 改成 b 的最低花销。

<small style="opacity: 0.7;"> 时间超级紧张，拼命写。但其实说实话感觉不是很贴切的笔试（???），很多类似小学竞赛题和趣味杂志上的题目。比较喜欢编程题，可惜样例有部分超时。</small>

#### 附加

解答题，给两个方框，写 md 格式回答。
- 一部 100000 字的书，怎么找出可能的错别字。
- 三部电梯，一些规则（诸如无人时怎么停法，有人叫时怎么开门，载人限制等等），一些参数，据此设计控制逻辑。

<small style="opacity: 0.7;"> 努力一本正经胡说八道，写了很多字（和代码）进去。一开始邮件通知挂了，后来 HR 又电话来说过了约时间，后来面试没参加。</small>
